# DECISIONS — 重要决策记录

> 记录产品和技术上的重要取舍。按时间倒序排列。

---

## 决策 #1：项目从零初始化

- **日期**：2025-07-07
- **决策**：当前仓库为空，使用 Next.js App Router + TypeScript + Tailwind CSS 从零搭建
- **原因**：无已有代码需要兼容，可以直接使用最新稳定技术栈
- **影响**：Phase 0 包含项目初始化（Git、Next.js 基础配置），Phase 1 开始编写组件
- **替代方案**：无（仓库为空，无替代）

## 决策 #2：品牌名称暂缓确定

- **日期**：2025-07-07
- **决策**：品牌名称待后续确定，代码中使用 "File Upload Assistant" 作为中性占位
- **原因**：避免在未进行品牌调研、域名检查、商标查询前锁定名称
- **影响**：所有代码中使用中性名称，后续统一替换成本低
- **替代方案**：先随意取一个名称 → 被否决，因为改名成本高且可能侵权

## 决策 #3：首版不引入数据库

- **日期**：2025-07-07
- **决策**：P0 阶段不使用任何数据库，所有处理在浏览器端完成
- **原因**：核心功能均为浏览器端文件处理，无需持久化存储；用户账号和云端保存属于 P2
- **影响**：架构简单、隐私风险低、无服务器存储成本；但无法提供历史记录功能（P1 考虑 IndexedDB 本地存储）
- **替代方案**：使用轻量数据库 → 暂不需要，P2 再评估

## 决策 #4：HEIC 支持推迟到 P1

- **日期**：2025-07-07
- **决策**：首版不包含 HEIC 转换功能
- **原因**：HEIC 浏览器兼容性不稳定，部分浏览器不支持解码。需要先验证依赖稳定性
- **影响**：P0 中遇到 HEIC 文件时显示"格式暂不支持"并提供明确说明
- **替代方案**：使用服务器端转换 → 被否决，违反本地处理优先原则

## 决策 #5：广告组件预留但不加载

- **日期**：2025-07-07
- **决策**：Phase 0-7 中预留 AdSense 代码架构，但默认关闭
- **原因**：AdSense 审核需要真实内容和稳定网站，不是开发阶段的前置条件
- **影响**：创建占位组件和配置开关，开关关闭时不发送任何请求
- **替代方案**：上线后再加广告代码 → 被否决，预留架构成本低且避免后期大改

## 决策 #6：P1 的"粘贴要求"功能使用规则解析而非 AI

- **日期**：2025-07-07
- **决策**：解析用户粘贴的上传要求时，第一版使用可解释的规则解析
- **原因**：不依赖付费 AI 接口；规则解析可解释、可调试、成本为零
- **影响**：解析能力受限于预设规则，复杂/非标准表述可能无法识别；需用户确认解析结果
- **替代方案**：使用 GPT API → 被否决，违反"不依赖付费 API"原则且增加隐私风险

## 决策 #7：使用 pdf-lib 处理 PDF

- **日期**：2025-07-07
- **决策**：浏览器端 PDF 处理首选 `pdf-lib`
- **原因**：纯 JavaScript 实现、无外部依赖、支持合并/拆分/旋转/页面操作、包体积可控（~300KB gzipped）、维护活跃
- **影响**：PDF 渲染/预览可能需要额外方案（pdfjs-dist 用于预览）
- **替代方案**：pdfjs-dist 全套 → 包体积过大；服务器端处理 → 违反本地处理原则

## 决策 #8：Phase 1 静态原型使用模拟数据

- **日期**：2025-07-07
- **决策**：Phase 1 中 File Compliance Checker 页面使用模拟数据和硬编码状态，不实现真实文件读取
- **原因**：本阶段目标只是验证界面和交互流程，真实文件处理将在 Phase 2 开始
- **影响**：页面可以点击按钮触发状态变化，但不会真正读取用户文件
- **替代方案**：Phase 1 就实现完整处理 → 违反分阶段开发原则

## 决策 #9：使用 Tailwind CSS v4 原生 CSS @theme 配置设计令牌

- **日期**：2025-07-07
- **决策**：使用 Tailwind CSS v4 的原生 CSS `@theme inline` 语法配置设计令牌，不使用传统 tailwind.config.js
- **原因**：Next.js 16 + Tailwind v4 的默认方式，更简洁
- **影响**：未来如需复杂主题切换，可能需要额外适配
- **替代方案**：使用 Tailwind v3 + tailwind.config.js → 额外维护负担

## 决策 #10：首页根路径 `/` 复用英文首页组件

- **日期**：2025-07-07
- **决策**：在 `/` 根路径渲染英文首页组件，`/[locale]` 路由通过复用该组件实现
- **原因**：Phase 1 先完成英文版；后续可扩展为基于 locale 的入口
- **影响**：Phase 6 多语言实现时需要重构根路由逻辑

## 决策 #11：压缩算法采用 Canvas + quality 二分搜索

- **日期**：2025-07-07
- **决策**：图片压缩优先使用 Canvas 导出 + 对 JPEG/WebP quality 做二分搜索，必要时回退尺寸缩小
- **原因**：浏览器原生支持、无额外依赖、效果好；PNG 为无损格式，压缩空间有限
- **影响**：压缩结果可能略大于目标值；极小目标可能需要尺寸缩小；PNG 透明转 JPEG 会变白底
- **替代方案**：使用 wasm 压缩库（如 mozjpeg）→ 包体积大、加载慢；服务器压缩 → 违反本地处理原则

## 决策 #12：OffscreenCanvas 为主线程 Canvas 的 fallback

- **日期**：2025-07-07
- **决策**：优先使用 OffscreenCanvas，不可用时 fallback 到 HTMLCanvasElement
- **原因**：Safari 不支持 OffscreenCanvas，必须兼容
- **影响**：代码稍复杂，但覆盖所有目标浏览器
- **替代方案**：只用 HTMLCanvasElement → 大图片可能阻塞主线程；使用 Web Worker + OffscreenCanvas → Safari 不支持
- **替代方案**：根路径直接重定向到 `/en` → 对 SEO 更友好，但 Phase 1 先保持简单
