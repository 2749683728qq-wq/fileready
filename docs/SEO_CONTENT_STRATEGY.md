# SEO_CONTENT_STRATEGY — SEO 与内容策略

> Phase 0 初稿 | 内容标准和 SEO 技术要求

## 1. 内容原则

- 内容首先服务真实用户，而不是为了堆关键词
- 每篇内容解决一个具体用户问题
- 不批量生成、不复制竞争网站、不虚构

## 2. 禁止事项

- 批量生成数百个近似页面
- 每个 KB 数字复制一个几乎相同的页面
- 复制竞争网站内容
- 自动改写官方要求
- 没有新增价值的聚合内容
- 虚构作者经验、测试结果、用户评价
- 用 AI 批量生成后未经人工检查直接发布
- 虚构评分、评论、价格的结构化数据

## 3. 首版内容目标（申请 AdSense 前的内部门槛）

| 类型 | 数量 | 说明 |
|------|------|------|
| 稳定工具 | 8-10 个 | P0 全部完成 |
| 场景页面 | 5-8 个 | 覆盖求职/学校/考试/签证/政府/办公 |
| 原创指南 | 12-20 篇 | 每篇解决一个具体问题 |
| 政策页面 | 完整 | 关于/联系/隐私/Cookie/条款/免责/无障碍/广告披露 |
| 质量要求 | — | 无空白页、无重复内容、导航和内链有效 |

## 4. SEO 技术要求

### 4.1 每页必须

- 独立 title（含品牌名）
- 独立 meta description（150-160 字符）
- 唯一 H1
- 语义化标题层级（H1 > H2 > H3）
- Canonical URL
- hreflang（双语）
- Open Graph（title, description, image, url）
- 面包屑 + BreadcrumbList 结构化数据

### 4.2 全站

- XML Sitemap（自动生成，排除 noindex 页面）
- robots.txt
- 正确 404 状态码
- 301 重定向规则
- 不索引参数页面和临时状态
- 服务端可读取的正文内容（非纯 JS 渲染）

### 4.3 结构化数据

可以使用：
- `WebSite`
- `Organization`
- `BreadcrumbList`
- 指南文章使用 `Article`

禁止：虚构评分、评论、价格或与页面内容不一致的标记。

## 5. 指南文章质量标准

每篇指南至少包含：
1. 用户具体问题（标题直接回应搜索意图）
2. 适用场景
3. 可执行步骤（非模糊建议）
4. 示例（前后对比数据）
5. 常见错误
6. 文件质量或隐私提醒
7. 相关工具链接
8. 更新时间
9. 内容审核状态

不以固定字数为标准，以是否完整解决问题为准。

## 6. 国家/机构文件要求（P1）

发布前必须：
- 记录官方来源 URL
- 记录最后核对日期
- 区分数字版和纸质版要求
- 标记工具可检查和不可检查的项目
- 无法确认时不得发布
- 不得声称代表官方
- 要求可能变化时显示提醒

## 7. 草稿内容管理

内容未完成时：
- 使用草稿状态
- 不加入导航
- 不加入 sitemap
- 设置 `noindex`
- 不展示广告

## 8. 多语言 SEO

- 独立 URL（`/en/...` `/zh-CN/...`）
- 自引用 canonical
- 双向 hreflang
- `x-default` 设置
- 不根据 IP 强制跳转
- 不把未翻译内容伪装成完整语言页面
- 不批量机器翻译后直接发布
- 每个语言版本的标题和描述独立检查
