# PRD — 产品需求文档

> 版本：0.1 | Phase 0 初稿 | 品牌名称待确定

## 1. 产品概述

### 1.1 产品名称（暂定）

**File Upload Assistant**（品牌名称待确定，代码中使用此中性占位名）

### 1.2 产品类别

在线文件上传处理与合规检查平台

### 1.3 一句话定位

帮助用户检查、压缩、调整、转换和整理图片、PDF、签名及扫描文件，使其更符合学校、求职、考试、签证、政府服务和其他在线系统的上传要求。

### 1.4 核心价值

```
上传文件 → 读取文件属性 → 对照用户要求检查 → 清楚说明不合格原因
→ 推荐可执行的修正方式 → 一键或分步修正 → 再次检查
→ 显示修改前后差异 → 下载
```

## 2. 目标用户

1. 求职者
2. 学生和学校申请者
3. 考试报名用户
4. 签证、护照及海外申请者
5. 使用政府和公共服务网站的人
6. 需要提交银行、保险或公司材料的人
7. 日常办公用户
8. 不熟悉图片、像素、DPI 和 PDF 技术术语的普通用户

## 3. 核心使用场景

| 场景 | 典型需求 |
|------|----------|
| 求职 | 压缩简历 PDF、合并简历与求职信、转换个人照片、调整签名、整理证明材料 |
| 学校和考试 | 报名照处理、签名尺寸调整、成绩单转 PDF、多份材料合并、文件大小检查 |
| 签证和护照 | 照片尺寸检查、图片比例调整、文件格式与大小检查、护照扫描件压缩、申请材料合并 |
| 政府和公共服务 | 附件格式检查、扫描件方向修正、PDF 页面处理、图片隐私信息清除、文件名安全检查 |
| 日常办公 | 邮件附件压缩、图片转 PDF、PDF 合并和拆分、HEIC 转换、图片尺寸调整 |

## 4. 功能范围

### 4.1 P0（首版必须完成）

1. **文件上传合规检查器**：支持图片和基础 PDF 检查，自定义要求参数
2. **图片压缩到指定大小以内**：JPG/JPEG/PNG/WebP，预设 + 自定义目标
3. **图片裁剪和尺寸调整**：自由裁剪、固定比例、数值输入、键盘微调
4. **图片格式转换**：JPG ↔ PNG ↔ WebP
5. **签名图片处理**：裁剪空白、尺寸调整、黑白/灰度、透明背景
6. **图片转 PDF**：单张/多张、A4/Letter/自适应、页面方向和边距
7. **PDF 合并**：多文件上传、拖拽排序、显示页数、本地处理
8. **PDF 拆分、提取和旋转**：指定页提取、页码范围、删除页、旋转
9. **图片元数据清理**：显示元数据类别、清除 EXIF、清除定位数据
10. **DPI 与尺寸换算器**：像素/毫米/厘米/英寸/DPI 换算

### 4.2 P1（稳定首版后开发）

- HEIC 到 JPG 或 PNG 转换
- 基础 PDF 体积优化
- 扫描件灰度和对比度增强
- 粘贴上传要求并提取限制（规则解析，不依赖付费 AI）
- 批量图片处理
- 常用要求预设
- 本地历史记录
- 国家或机构文件要求数据库

### 4.3 P2（暂不开发）

- 用户账号、云端文件保存、会员和支付
- OCR、Word 和 Excel 复杂转换
- 电子签名服务、在线协作
- 自动提交申请、复杂 AI 人脸判断
- 大规模国家模板库

## 5. 产品边界

本产品只处理文件技术属性，不提供：
- 签证、移民、法律、医疗、税务或行政审批建议
- "保证申请通过"或"保证文件一定被机构接受"
- 政府或官方机构身份暗示
- 证件伪造、修改身份信息、修改证件内容
- 人脸重塑或欺骗性照片处理
- 绕过平台安全检查、破解 PDF 密码、移除不属于用户的版权保护

所有相关页面需说明：**工具只能检查和处理可读取的文件技术属性，最终要求以接收文件的机构或平台公布的信息为准。**

## 6. 用户体验核心原则

- 首页不展示几十个工具，首屏询问"你的文件遇到了什么问题？"
- 主要按钮唯一且突出："检查我的文件"
- 每个工具实现 16 种状态（不只是理想成功流程）
- 处理完成后显示前后对比
- 下载按钮附近不得出现广告
- 所有文案首先服务真实用户，不是为了堆关键词

## 7. 视觉风格

- **关键词**：简洁、清晰、可信、安全、现代、克制、轻量、面向普通用户
- **目标感受**：像一个可靠、安静、专业的在线服务，而不是廉价 SEO 工具站
- **禁止**：大面积渐变、玻璃拟态、霓虹颜色、过多阴影、满屏动画、自动播放视频、用表情符号代替正式图标
- **颜色方向**：主色为可信蓝色、辅助色为少量青绿色、页面背景为非常浅冷灰色、卡片白色

## 8. 多语言策略

- 首版：英文 + 简体中文
- 预留日文（未完成专业校对前不发布）
- 独立 URL、正确 hreflang、不根据 IP 强制跳转、不批量机器翻译

## 9. 技术约束

- Next.js App Router + TypeScript 严格模式 + Tailwind CSS
- 文件处理优先浏览器端（File API、Canvas、Web Worker）
- 不将用户文件发送到服务器
- 不依赖付费 AI 接口
- 不暴露密钥
- 设置合理 CSP 和安全响应头
