# DESIGN SYSTEM — 设计系统初稿

> Phase 0 初稿 | 具体色值待视觉细化确认

## 1. 设计原则

- **清晰优先**：信息层级分明，操作意图明确
- **可信可靠**：不花哨、不浮夸、不伪装
- **克制轻量**：用最少的设计元素表达必要的信息
- **可访问**：WCAG 2.2 AA 为目标，对比度达标、键盘可操作
- **面向普通用户**：不假设用户理解技术术语

## 2. 颜色系统

### 2.1 主色调

| 令牌 | 用途 | 建议色值 | 说明 |
|------|------|----------|------|
| `primary-600` | 主按钮、链接、焦点 | `#2563EB` | 清晰可信的蓝色 |
| `primary-700` | 主按钮悬停 | `#1D4ED8` | |
| `primary-50` | 主色浅背景 | `#EFF6FF` | 选中/高亮背景 |
| `accent-500` | 辅助状态 | `#14B8A6` | 少量青绿色 |
| `accent-50` | 辅助浅背景 | `#F0FDFA` | |

### 2.2 中性色

| 令牌 | 用途 | 建议色值 |
|------|------|----------|
| `gray-50` | 页面背景 | `#F8FAFC` |
| `gray-100` | 卡片/输入框背景 | `#F1F5F9` |
| `gray-200` | 边框 | `#E2E8F0` |
| `gray-400` | 占位文字 | `#94A3B8` |
| `gray-500` | 次要文字 | `#64748B` |
| `gray-700` | 主要文字 | `#334155` |
| `gray-900` | 标题文字 | `#0F172A` |
| `white` | 卡片/模态背景 | `#FFFFFF` |

### 2.3 语义色

| 令牌 | 用途 | 建议色值 |
|------|------|----------|
| `success-600` | 通过状态 | `#16A34A` |
| `success-50` | 通过浅背景 | `#F0FDF4` |
| `warning-600` | 警告/需确认 | `#EA580C` |
| `warning-50` | 警告浅背景 | `#FFF7ED` |
| `error-600` | 未通过/错误 | `#DC2626` |
| `error-50` | 错误浅背景 | `#FEF2F2` |

### 2.4 使用规则

- 统一使用 CSS 变量或 Tailwind 设计令牌，不在业务页面直接写十六进制颜色
- 所有文字与背景组合必须验证对比度（≥4.5:1 正文，≥3:1 大文字）
- 不只依靠颜色区分状态，同时使用图标或文字

## 3. 字体

### 3.1 英文

- 使用 `next/font` 自托管清晰无衬线字体（建议 Inter 或类似）
- 不得从外部 CDN 运行时加载字体

### 3.2 中文/日文

- 使用稳定系统字体栈：`'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', sans-serif`
- 日文预留：`'Hiragino Kaku Gothic ProN', 'Noto Sans CJK JP', sans-serif`

### 3.3 字号规范

| 层级 | 桌面 | 移动 | 用途 |
|------|------|------|------|
| H1 | 32px/40px | 24px/32px | 页面标题 |
| H2 | 24px/32px | 20px/28px | 区块标题 |
| H3 | 20px/28px | 18px/24px | 子标题 |
| Body | 16px/24px | 16px/24px | 正文（不小于 16px） |
| Body-sm | 14px/20px | 14px/20px | 辅助文字 |
| Caption | 12px/16px | 12px/16px | 标签/备注（不得更小） |

## 4. 间距系统

统一使用 4px 基准间距：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`

禁止使用无规律的 13px、19px、27px 等数值。

## 5. 圆角

| 元素 | 圆角 |
|------|------|
| 小控件（标签、徽章） | 8px |
| 输入框、按钮 | 12px |
| 普通卡片 | 16px |
| 大型上传区域 | 20px |

## 6. 阴影

- 默认优先通过边框和背景区分层级
- 阴影只用于确实需要抬升的元素（如对话框、下拉菜单）
- 不允许每张卡片都有明显阴影
- 阴影定义（Tailwind 风格）：
  - `sm`：`0 1px 2px rgba(0,0,0,0.05)`
  - `md`：`0 4px 6px -1px rgba(0,0,0,0.07)`
  - `lg`：`0 10px 15px -3px rgba(0,0,0,0.08)`

## 7. 图标

- 使用统一线性图标库（建议 Lucide Icons）
- 图标必须有一致尺寸（20px/24px）和笔画宽度
- 图标按钮必须有可访问名称（aria-label）
- 不混用多个图标库
- 不用表情符号代替正式图标

## 8. 动画

- 时长控制在 120-200ms
- 只用于说明状态变化（hover、展开、切换）
- 不得延迟用户操作
- 尊重 `prefers-reduced-motion`
- 文件处理不使用虚假进度条
- 过渡缓动：`ease-out`（进入）、`ease-in`（退出）

## 9. 组件命名约定

组件文件使用 PascalCase，按功能分组：

```
components/
  ui/           # 基础 UI 组件（Button, Input, Select...）
  layout/       # 布局组件（Header, Footer, Sidebar...）
  tools/        # 工具专属组件
  shared/       # 跨工具共享组件
  ads/          # 广告相关组件
  seo/          # SEO 相关组件
```

## 10. 响应式断点

| 断点 | 宽度 | 用途 |
|------|------|------|
| `sm` | 640px | 大手机横屏 |
| `md` | 768px | 平板 |
| `lg` | 1024px | 小桌面 |
| `xl` | 1280px | 桌面 |
| `2xl` | 1536px | 大桌面 |

移动优先设计。工具正文区桌面端可使用较窄阅读宽度（最大 720px）。
