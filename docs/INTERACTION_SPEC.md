# INTERACTION SPEC — 交互规范

> Phase 0 初稿 | 工具交互流程和状态规范

## 1. 通用交互流程

每个核心工具必须完整实现以下 16 种状态：

| # | 状态 | 说明 |
|---|------|------|
| 1 | 初始状态 | 工具入口，操作指引清晰可见 |
| 2 | 拖拽悬停 | 文件拖入上传区域时的视觉反馈 |
| 3 | 文件已选择 | 显示文件名、类型、大小等基本信息 |
| 4 | 文件读取中 | 正在读取文件属性 |
| 5 | 文件读取成功 | 显示详细文件信息，激活参数设置 |
| 6 | 文件格式不支持 | 明确说明不支持的原因，提供替代建议 |
| 7 | 文件超限 | 文件超过浏览器处理限制时的提示 |
| 8 | 参数不合法 | 用户输入参数超出允许范围 |
| 9 | 正在处理 | 处理进度（非虚假进度） |
| 10 | 用户取消 | 取消处理，恢复可操作状态 |
| 11 | 处理成功 | 显示前后对比，提供下载 |
| 12 | 处理失败 | 保留原文件，说明失败原因，提供恢复建议 |
| 13 | 重新调整 | 从结果页返回调整参数 |
| 14 | 下载成功 | 下载触发的确认反馈 |
| 15 | 用户更换文件 | 清除当前文件，回到初始或选择状态 |
| 16 | 低内存/不兼容 | 检测到浏览器不支持或内存不足 |

## 2. 文件上传区域

### 2.1 拖拽区域规范

- 明显可见的拖拽区域（虚线边框 + 浅背景）
- 拖拽悬停时边框高亮 + 背景变化
- 同时支持点击打开文件选择器
- 显示支持的文件格式和大小限制
- 移动端同样支持（点击选择 + 拍照选项）

### 2.2 文件选择后信息展示

```
┌──────────────────────────────────────┐
│ 📄 文件名.jpg  (截断超长名称...)    │
│ 类型: JPEG  |  大小: 1.2 MB        │
│ 尺寸: 1920 × 1080  |  方向: 横向   │
│                                      │
│ [更换文件]  [删除]                   │
│                                      │
│ ℹ️ 文件在你的浏览器中处理           │
└──────────────────────────────────────┘
```

- 长文件名截断但允许点击查看完整名称
- 显示可读取的所有技术属性
- 本地处理说明文字明显但不过度突出

## 3. 处理结果页

### 3.1 前后对比

```
┌───────────────────────────────────────┐
│          处理完成                      │
│                                       │
│  处理前          →       处理后       │
│  ┌─────────┐          ┌─────────┐    │
│  │ 预览图  │          │ 预览图  │    │
│  └─────────┘          └─────────┘    │
│                                       │
│  文件大小: 2.4 MB  →  180 KB        │
│  格式:     JPEG    →  JPEG          │
│  尺寸:     4000×3000 → 1920×1440   │
│  宽高比:   4:3     →  4:3 (保持)   │
│                                       │
│  [下载处理后的文件]  ← 主按钮        │
│  [重新调整]  [处理另一个文件]        │
└───────────────────────────────────────┘
```

### 3.2 下载按钮

- 主按钮突出、唯一
- 附近不得出现广告
- 下载前给予清晰的文件名和格式提示
- 支持"另存为"自定义文件名（可选）

## 4. 错误处理

### 4.1 错误信息要求

- 用普通语言说明问题（非技术术语）
- 说明具体原因
- 提供可执行的解决建议
- 错误信息与相关字段关联（aria-describedby）
- 错误状态使用图标 + 颜色 + 文字三重区分

### 4.2 常见错误类型

| 错误 | 用户提示示例 |
|------|-------------|
| 文件太大 | "此文件为 25 MB，超过浏览器处理限制 20 MB" |
| 格式不支持 | "不支持 HEIC 格式，你可以先转换为 JPG 后再试" |
| 损坏文件 | "无法读取此文件，文件可能已损坏" |
| 加密 PDF | "此 PDF 已加密，请先解除密码保护后再试" |
| 处理失败 | "压缩失败，可能是图片质量已经很低。建议尝试..." |

## 5. 键盘操作

- Tab 键在可操作元素间移动
- Enter/Space 激活按钮
- Escape 关闭对话框/下拉
- 裁剪工具支持方向键微调（±1px/±10px）
- 焦点样式清晰可见（2px 外框，与主色一致）
- 拖拽排序支持键盘操作（Space 提起、方向键移动、Space 放下）

## 6. 加载和进度

- 不使用虚假进度条（匀速增长但实际未完成）
- 短操作使用旋转图标
- 长操作显示实际处理阶段（"读取文件中..." → "压缩中..." → "生成下载..."）
- 可取消的操作提供取消按钮
- 超过 10 秒的操作给出预估时间提示

## 7. 移动端适配

- 单栏布局
- 触控目标至少 44×44 CSS 像素
- 裁剪工具提供放大预览辅助精确操作
- 文件选择支持调用相机（accept="image/*" capture）
- 长列表使用虚拟滚动或分页

## 8. 文案规范

- 使用主动、清晰的动词（"下载"而非"进行下载"）
- 按钮文案描述结果（"压缩图片"而非"提交"）
- 不使用技术术语吓唬用户（说"图片太大"而非"超出最大像素限制"）
- 中英文混排时在英文单词前后加空格
- 数字使用千位分隔符（1,024 KB 而非 1024 KB）
